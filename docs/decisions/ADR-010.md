# ADR-010: Advanced Agentic RAG System Upgrade

- **Date:** 2026-03-14
- **Status:** Accepted
- **Deciders:** CropFresh AI Team

## Context

CropFresh AI's RAG system has functional components (RAPTOR indexing, hybrid search, CRAG grading, 8-strategy router, speculative drafting, self-evaluation) but lacks critical anti-hallucination mechanisms and real evaluation infrastructure. Farmers rely on AI-generated advice for pesticide doses and market decisions — hallucinated answers are safety-critical.

## Decision

Implement a 4-phase upgrade:

1. **Anti-Hallucination Pipeline** — `query_rewriter.py`, `citation_engine.py`, `confidence_gate.py`
2. **RAGAS Evaluation + CI Gate** — Replace mock evaluation, 30 golden queries, fail deployment if faithfulness < 0.80
3. **Advanced Retrieval** — Contextual chunking, parent-child retrieval, time-aware hybrid search
4. **LangGraph State Machine** — Replace imperative `orchestrate()` loop with declarative LangGraph graph

## Consequences

### Positive
- Zero-hallucination guarantee via confidence gating + citation verification
- Measurable RAG quality via real RAGAS evaluation
- Declarative, testable retrieval flow via LangGraph
- Self-corrective: incorrect answers trigger automatic retry with query rewriting

### Negative
- Phase 4 breaks internal orchestrator implementation (public API preserved)
- ~2,300 new lines of code to maintain
- Evaluation pipeline adds ~30 seconds to CI

### Neutral
- No AWS infrastructure changes (all pure Python within `ai/rag/`)
- No new dependencies (langgraph, ragas already in `pyproject.toml`)
